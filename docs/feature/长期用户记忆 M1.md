# 长期用户记忆 M1

## 背景与目标

- Session history 和 working memory 已分别覆盖当前会话原文历史和会话级结构化状态。
- 长期用户记忆用于保存同一用户跨会话可复用的低风险偏好，例如连接方式、预算、用途和品牌偏好。
- 本阶段不依赖完整用户中心，只依赖认证后的 `current_user.id`，并继续挂载到 `memory_fact.user_id`。

## 实现范围

- 为 `memory_fact` 增加治理字段：
  - `scope`
  - `fact_type`
  - `expires_at`
  - `last_used_at`
  - `disabled_at`
- `ConversationRepository.list_memory()` 只返回当前用户、未禁用、未过期的记忆。
- `ConversationRepository.upsert_memory()` 写入 `scope`、`fact_type`、`source_message_id` 和 `last_used_at`。
- 新增 `ConversationRepository.disable_memory()`，用于按当前用户禁用记忆。
- `MemoryService.extract_long_term_facts()` 从用户消息中提取低风险偏好。
- Agent 在 `in_scope_auto` 请求结束时写入长期用户记忆，来源指向当前用户消息 ID。

## 隔离规则

- 只按认证后的 `current_user.id` 读写长期记忆。
- 不接受请求体 `user_id`。
- 长期记忆可以跨同一用户的多个会话读取。
- 长期记忆不能跨用户读取。
- `disabled_at` 不为空的记忆不进入 Agent context。
- `expires_at` 已过期的记忆不进入 Agent context。

## 安全边界

- 当前只抽取低风险偏好：
  - 无线/有线偏好
  - 预算偏好
  - 游戏/办公用途
  - 常见品牌偏好
- 当前不写入手机号、地址、完整物流单号、真实售后责任结论、退款承诺或其他敏感信息。
- 后续若引入 mem0 或其他外部记忆系统，应通过 `LongTermMemoryProvider` adapter 接入，不能绕开本地用户隔离和治理字段。
